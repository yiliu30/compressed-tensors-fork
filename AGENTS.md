# AGENTS.md

This file provides guidance to AI coding agents when working with code in this repository.

## Project Overview

`compressed-tensors` is a library by the vLLM project that extends safetensors to provide a unified format for storing and loading compressed (quantized/sparse) neural network models. It supports GPTQ, AWQ, SmoothQuant, INT8, FP8, SparseGPT, and more. It integrates with Hugging Face Transformers via `CompressedTensorsHfQuantizer`.

## Architecture

### Source Layout (`src/compressed_tensors/`)

The package uses `src/` layout with `package_dir={"": "src"}`.

### Registry Pattern

The `RegistryMixin` (`registry/registry.py`) is a central pattern used throughout. Classes inherit from `RegistryMixin` and use the `@MyClass.register()` decorator to register implementations. Lookup names are standardized (lowercase, hyphens replace underscores/spaces). Supports aliases and loading from external module paths via `"path/to/module.py:ClassName"` syntax.

Key registry-based base classes:
- `BaseCompressor` — compression format implementations
- `TransformFactory` — transform implementations (Hadamard, random Hadamard, matrix multiply)
- `SparsityCompressionConfig` — sparsity config types

### Quantization Lifecycle

Quantization follows an ordered lifecycle tracked by `QuantizationStatus`:
`INITIALIZED → CALIBRATION → FROZEN → COMPRESSED → DECOMPRESSED`

Key flow:
1. **apply_quantization_config()** (`quantization/lifecycle/apply.py`) — attaches `QuantizationScheme` to matched modules, initializes quantization parameters (scales, zero points)
2. **Calibration** — observers collect statistics during forward passes
3. **freeze_module_quantization** — removes observers, freezes parameters
4. **compress_quantized_weights** / `ModelCompressor.compress_model()` — compresses weights in-place
5. **Decompression** happens automatically on forward pass or explicitly via `decompress_model()`

### Compression Formats & Compressors (`compressors/`)

`BaseCompressor` subclasses implement `compress()` / `decompress()` as classmethods operating on per-module state dicts (local parameter names like `weight`, `weight_scale`). Format selection uses a priority list in `compressors/format.py` — each compressor's `can_compress()` determines applicability.

Formats: `dense`, `naive-quantized`, `int-quantized`, `float-quantized`, `pack-quantized`, `nvfp4-pack-quantized`, `mxfp4-pack-quantized`, `mixed-precision`. Legacy formats: `sparse-bitmask`, `sparse-24-bitmask`, `marlin-24`.

### ModelCompressor (`compressors/model_compressors/`)

Orchestrates full model compression/decompression. Two entry points:
- `ModelCompressor.from_pretrained_model(model)` — used by llm-compressor when saving
- `ModelCompressor.from_compression_config(ct_config)` — used by HF Transformers when loading

### Quantization Config (`quantization/`)

Pydantic models define the configuration hierarchy:
- `QuantizationConfig` — top-level config with `config_groups` mapping group names to `QuantizationScheme`
- `QuantizationScheme` — per-group settings with targets, weights/input/output `QuantizationArgs`
- `QuantizationArgs` — per-tensor quantization parameters (num_bits, type, strategy, group_size, symmetric, dynamic)

Preset schemes (e.g., "W8A8", "W4A16") are resolved in `preset_name_to_scheme()`.

### Transforms (`transform/`)

Activation/weight transforms (e.g., Hadamard rotations for SmoothQuant-style methods). `TransformFactory` subclasses create and apply transforms. Weight transforms are fused into weights; activation transforms are attached as submodules via PyTorch hooks.

### Offload System (`offload/`)

Manages CPU/disk offloading of model parameters for memory-efficient processing. Uses `OffloadCache` replacing `module._parameters` and `module._buffers`. Key functions: `update_offload_parameter()`, `align_module_device()`, `dispatch_model()`. Context managers `disable_offloading()` / `disable_onloading()` control behavior.

### Config Types (`config/`)

`CompressionFormat` enum and `SparsityCompressionConfig` for sparsity-specific settings (format, targets, ignore lists, sparsity structure like "2:4" or "unstructured").

### Modeling (`modeling/`)

Attention hooking (`attention.py`) and KV cache quantization (`kvcache.py`) — used to intercept and quantize keys/values in transformer attention layers.

## Test Markers
